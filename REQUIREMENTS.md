# Projet Xamarin - Requirements

This document outlines the key requirements and functionalities for the "Projet Xamarin" contact management application, based on the provided assignment specifications.

## 📱 Core Features (Screens & Actions)

- [ ] **Contact List**: Display a list of all contacts.
- [ ] **Alphabetical Sorting**: The contact list must be sorted in alphabetical order.
- [ ] **Create Contact**: A screen/form to add a new contact.
- [ ] **Edit Contact**: A screen/form to modify an existing contact's details.
- [ ] **Delete Contact**: The ability to remove a contact from the list.

## 👤 Contact Data Model

Each contact must store the following information:
- [ ] First Name (*Prénom*)
- [ ] Last Name (*Nom*)
- [ ] Mobile Number (*Numéro de GSM*)
- [ ] Email Address
- [ ] Comments/Notes (*Commentaire*)

## 💾 Technical Requirements

- [ ] **Data Persistence**: The app must save data locally (e.g., using a database or files) so that contacts survive an app restart.
- [ ] **Cross-Platform**: Built using Xamarin / .NET.

## 🌟 Bonus Features (Extra Points)

- [ ] **Profile Pictures**: The ability to add a photo to a contact.
- [ ] **Cloud Sync**: Implement an online synchronization system (e.g., using Sockets, REST API, etc.) to back up or sync contacts online.

---
*Note: The project requires an oral presentation to demonstrate these functionalities. You must submit the source code and all files necessary to build/run the application.*
